지금 만들고 있는 GUI/스크립트의 최종 목표는 단순 통계 뷰어가 아니라 **개인 전투로그 진단 도구** (= 스펙 코치) 다.

**최종 사용 흐름:**
1. 사용자가 본인 WCL 리포트 URL/코드 + 캐릭터 + 보스를 입력
2. 도구가 본인 로그의 (특성 / 아이템 / 캐스트 타임라인 / 버프 이벤트) 을 긁어와서
3. 같은 보스 같은 스펙의 **top100 레퍼런스** 와 차이를 진단:
   - 특성: top100 선택 분포 vs 본인 → 어떤 노드가 의외로 비주류/주류
   - 아이템: top100 장비/장식/보석/특보 vs 본인 → 빠진 거, 잘못 박은 거
   - 딜사이클: 핵심 명령기 시전 빈도/타이밍 vs 본인 → 빠뜨린 시전, 잘못 끼워넣은 시전
   - 물약: top100이 쓰는 타이밍(주로 풀ldata버스트) 에 본인이 안 썼는가
   - 쿨기: top100이 돌리는 시점·간격 vs 본인 → 못 돌린 거, 늦게 돌린 거
   - 프록/추적버프: 발동된 프록 → 다음 시전 매칭이 잘 됐는가

**범위:**
- 1차 타깃 스펙 5종: 악마흑마 (Demonology Warlock), 조화드루이드 (Balance Druid), 야수냥꾼 (Beast Mastery Hunter), 무기전사 (Arms Warrior), 분노전사 (Fury Warrior)
- 1차 타깃 존: 46 (한국 클라 "꿈의 균열" 레이드, 신화 우선)

**표준 로테 비교 (예정 — 미구현):**
- 타임라인에서 본 캐릭의 시전이 IcyVeins/WoWhead 표준 로테와 다른 부분을 짚어주는 기능
- 필요한 것:
  - 스펙별 표준 로테/우선순위 인코딩 (악마/조화/야수 3스펙 우선)
  - 출처는 IcyVeins (예: icy-veins.com/wow/demonology-warlock-pve-dps-rotation-cooldowns-abilities)
    및 WoWhead 가이드 (wowhead.com/guide/classes/...)
  - 표준이 단순 sequence 가 아니라 "X 발동 시 Y", "쿨 정렬 시 Z" 같은 priority rule 임
- 차이 진단:
  - 핵심 쿨다운 미사용 / 늦게 사용
  - 프록 (예: 악마 핵) 발동 후 매칭 안 된 시전
  - 추적버프 (예: 일식) 상태에서 잘못된 시전
  - 물약 비사용 / 잘못된 타이밍

**관측해야 할 축 (cast 단독으로는 부족):**
- 캐스트 타임라인 (in progress: fetch_casts.py)
- 쿨다운 사용 (대형 CD: 야성의 격노/천계 정렬/소환 폭군 등)
- 물약 사용
- 개인 생존기 사용
- **프록**: 발동된 버프(예: 악마 핵, 별빛섬광 프록, 광폭화)와 그걸 어떻게 소비했는지
- **추적버프**: 일식/스타로드/광폭화 누적 같은 상태 버프가 시점별로 어떻게 유지/소진되었는지
- **특성 트리 전체** (현재 cache_talents.json엔 picked nodes만 있음 — 시각적 트리 렌더는 WoWhead 위젯/URL 으로)
- **아이템 장비** (모든 슬롯 + 보석/마부/장식/특보 보너스ID) — 아직 캐시 없음, V2 `reportData.report.table(dataType: Summary)` 에서 같이 옴

**보스 페이즈/조 분기 처리 (중요):**
- 조 분기 신경 써야 하는 보스는 **카이메루스 (Chimaerus, the Undreamt God / 꿈의균열, encounter_id 3306) 하나뿐**. 나머지 8보스는 그룹 분기 무시 가능.
- 카이메루스는 보스의 특정 시전 시점에 1조/2조로 갈라짐 → 같은 시점이라도 두 그룹의 로테가 달라짐
- 로그의 캐스트 패턴/디버프 부여로 그 사람이 어느 조였는지 **추론**해서 1조용·2조용 사이클을 분리해 보여줘야 함
- 트리거 시전 정확한 스펠 ID는 아직 미확정 — 데이터 다 모인 뒤 보스 캐스트 로그 보고 사용자랑 매핑

**Why:** 단순 "이 사람이 뭘 시전했나" 가 아니라 "어떤 상황에서, 어떤 프록/추적버프 상태에서, 어떤 조에 속해서 어떤 선택을 했는가"를 학습용으로 보여주는 게 목적.

**How to apply:** 추가 기능 제안할 때 이 비전 안에서 우선순위 따져 제안. 단순 평균/CV/카운트류 통계는 이미 끝났음 — 다음 단계는 항상 "상황 라벨링 + 시계열" 방향.
