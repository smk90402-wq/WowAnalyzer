---
name: 자율적으로 진행
description: 명백한 follow-up 작업은 사용자에게 물어보지 말고 스스로 진행. 진짜 결정 필요한 것만 물어봄.
type: feedback
---

**규칙**: 작업 끝낼 때마다 follow-up todo (이전 turn에 언급됐거나 명백히 다음 단계인 것)는 사용자 지시 없이 바로 진행한다. AskUserQuestion 은 다음 경우에만:

1. **데이터 손실 / 비가역 작업**: API 재패치(수십분~수시간), 캐시 폐기, 기존 CSV 덮어쓰기
2. **취향 선택**: "라이브러리 A vs B", "다크 vs 라이트 테마", "스펙 우선순위 인코딩의 기준"
3. **두 갈래 길이 진짜 갈리는 경우**: 비용·이득이 비슷한데 사용자 선호가 결과를 가르는 경우
4. **요구사항이 모호한 경우**: 사용자 메시지를 다르게 해석할 수 있고, 잘못 짚으면 일 다시 해야 함

**Why:** "앞으로는 지시받지말고 할일있으면 알아서 하나하나 진행하도록해" — 사용자가 명시적으로 자율 모드 요청. 매 turn 마다 "다음 뭐 할까?" 묻는 게 흐름 끊고 짜증나는 듯.

**How to apply:**
- 이전 turn 에 "다음 후보" 로 적어둔 작업이 자율 처리 가능하면 → 그냥 진행
- API 비용 큰 작업 (재패치, 풀 enrich) 은 항상 ask
- 코드/리팩토링/UI 개선/메모리 갱신 은 자율로 진행
- 진행 후 turn 끝에 한 줄로 "다음 X 했음" 보고
