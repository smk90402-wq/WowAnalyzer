name: 매 turn 끝 상태 보고
description: 매 turn 끝에 진행 중/백그라운드/완료/빌드 상태를 표준 블록으로 출력
type: feedback
---

**규칙**: 매 응답 끝에 반드시 "## 상태" 섹션을 붙인다. 형식:

```
## 상태
- **이번 turn 완료**: <한 줄 요약 X개>
- **백그라운드 진행 중**: <jobID + 무엇 + 진행률 / 없으면 "없음">
- **빌드**: ✅ 완료 (Xs) / 🔄 진행 중 / ❌ 실패
```

**"다음 자율 작업" 줄은 절대 추가하지 않는다.** 사용자가 명시적으로 빼라고 함 ("왜 자꾸 알려주는거야? 스스로하면되지않아?"). 다음에 할 일이 있으면 그냥 그 turn 에 진행한다 — 사용자에게 의향 묻는 줄 X.

**Why:** 사용자 명시 — "채팅 끝날때마다 현재 진행중인 작업이있는지 지속적으로 알려주고, 백그라운드에서도 돌아가는거 있는지 알려주고 끝난 작업은 어떤작업이끝났다고 알려주고 빌드도 완료됐다고 지속적으로". 백그라운드 잡이 많아져서 상태 추적 어려움.

**How to apply:**
- 매 응답 끝에 status 블록 (위 형식 그대로)
- 백그라운드 job ID 와 마지막 로그 줄 둘 다 보여줘서 사용자가 직접 추적 가능하게
- "이번 turn 완료" 는 사용자가 보기에 의미 있는 단위 (스크립트 작성, 파일 수정, 메모리 갱신 등)
- "다음 자율 작업" = 자동으로 이어갈 일이 있으면 명시 (없으면 "대기")
- 사용자가 한 일을 다시 묻지 않도록 — status 만 봐도 전체 그림 파악 가능해야
