## 네임드별 실측 딜사이클 (extract_boss_dealcycle.py → boss_dealcycle.json)
**사용자 핵심 아이디어**: 가이드 일반로테 말고, top100이 보스별로 실제 어떻게 쳤나를
events 캐시(casts+buffs 타임스탬프)로 역산. 딜사이클 탭 '보스별 실측' 모드(토글).
추출: 오프너 합의(최빈 첫7캐스트)·쿨기 첫사용/횟수·블러드 커버리지+타이밍·물약·버프업타임·
빌드분기(회전베기 node 102341→광/단일)·킬타임. /api/boss-dealcycle.
- **데이터 한계 (정직표기)**: ① 블러드 — BM은 펫 원시격노라 100% 잡힘, MM/생존은 외부
  주술사 블러드라 받은판만 집계(targetID 캐시지만 그 판에 안받으면 없음). 커버리지 n/N 노출.
  ② 물약 추적 희박(6/25 수준) = 참고용. ③ 빌드자동감지가 우리가 찾은 우주의왕관 단일0% 등과
  일치 = 검증됨. 강한 항목: 오프너·쿨기타이밍·킬타임·버프업타임·빌드.
- 확장: SPEC_CONFIG 에 (class,spec) 쿨기/추적버프 추가하면 타직업도 추출. 현재 사냥꾼 3스펙.
- **상자(알게타르 수수께끼 383781) 타이밍** (사용자 요청): 오프닝(≤10s) 사용% + 미오프닝 첫사용
  중앙값. boss_dealcycle.json box필드, 카드 '상자' 행. **우주의왕관=오프닝 0%·~27s 첫사용**
  (전부 지연, 뎀증창 27s 추정 — 사용자 "뎀증타이밍 쿨기 못씀"과 연결). 벨로렌/카이메루스
  ~120s(2분쿨 버스트창). 보라시우스/아베르지안 등은 오프닝 사용(70~91%).
- ⚠️ **오프너 추출 버그 교훈 (사용자 지적)**: ① 위치별 최빈값(positional mode)은 가짜
  합성/중복 만듦 → **medoid**(나머지와 가장 유사한 실제 1명 시퀀스)로 교체. opener_match%=대표성.
  ② 1초내 연타(오프-GCD 펫프록·종족특성)가 GCD슬롯 먹는 노이즈 → **750ms(GCD최저치) 접기**.
  ③ 종족특성(피의 격노 20572)·버프프록(빛의 잠재력 1236616)·이동기(철수781·치타상)는
  cast로 찍혀도 딜로테 아님 → NOISE_CASTS 제외. 장신구(알게타르383781)는 유지.

## 스탯 탭 (extract_boss_stats.py → boss_stats.json) — 보스별 스탯 분포
**사용자 통찰 입증됨**: 같은 전문화도 광특/단일특 스탯 다름. 살라다르 실측: 광=치명1196/
가속398, 단일=치명910/가속683 (같은 보스·ilvl인데 빌드만 다르면 확 갈림). 야수 광=치명42%/
단일=가속27%.  탭 구성: 1~20등 개별(순위·DPS·빌드·각스탯·비중막대) + 21~100 빌드별평균(광/단일
분리). /api/boss-stats. player_fight.stats(특화/치명/가속/유연+ilvl). 막대색: 특화금·치명빨·
가속파·유연초. SPEC_BUILD 에 (class,spec,광노드) 추가하면 타직업 확장. ⚠ stats 캐시 커버리지
top20=14~20명 충분, 21~100=10~74명 들쭉(평균은 n 표기). 빌드분기는 회전베기 node 102341.

## 스탯 탭 추가기능 (사용자 요청 3가지)
- **수치+실효% 병기**: 막대(%)에 raw rating + 실효%(DR적용) 같이. boss-stats 엔드포인트가
  eff 계산해 내려줌. 특화는 spec별 환산이라 실효% 공란(정상).
- **스탯 점감(DR, data/stat_dr.json)**: maxroll.gg 한밤 12.0.1 2차스탯 점감(게임 메커니즘=사실).
  레벨90: 가속 1320(30%)까지 무점감, 초과분 누진 페널티(-10~-100%). 1%환산 가속44·치명46·유연54.
  _effective_pct() 가 누진계산 (maxroll 예시 1584=35.4% 검증일치). ★브래킷 경계=구간 *시작*임 주의.
- **캐릭터 장비창**: 스탯 top행 클릭 → **/api/gear (경량 전용)**. 보석=item_db 매핑(이름·아이콘
  완벽: 보석 ID는 아이템 ID라 item_db 에 있음). 마부=enchant ID(7961 등)는 별도체계라 이름DB
  없음 → **wowhead 파워툴팁(power.js)**으로 해결: 아이템에 data-wowhead="item=X&ench=Y&gems=Z"
  호버하면 wowhead 가 풀스탯+마부명+보석명 다 띄움. ⚠ 마부 ID 를 Blizzard item API 에 넣으면
  엉뚱한 동번호 아이템 잡힘(7961→"환영검" 오매핑) — 추측매핑 금지, wowhead 툴팁이 정답.
  ★성능: events 캐시(수백MB) lazy-load 로 분리(wcl_v2_data.py @property events). gear 는
  player_fight 만 봐서 첫클릭 30s→4.3s, 이후 17ms. boss_stats 엔 ref(rid/fid/char)만(228KB).
  빈슬롯(id=0) 제외. /api/character 는 events 페치라 느림 → gear 는 /api/gear 써야함.

## 스탯 탭 버프 기준 (사용자 질문 — 확정)
WCL combatantInfo.stats = **풀버프 스냅샷**(pull 시점). 검증결과:
- **% 버프(블러드·마력주입·표식)는 제외됨** — rating 기반이라 %는 안 더해짐.
- **전투 물약(열광의물약 등)도 제외** — 전투중 발동이라 스냅샷에 없음.
- **음식·영약·무기기름(flat rating)은 포함** — pull 시점 active. (단정증거: 같은캐릭 같은ilvl290
  인데 특화 1238~2137 ±900변동 = 음식/영약 고정값으론 불가 → 100% 장비교체. 즉 음식/영약은
  모든판 공통상수라 광/단일·보스간 비교 왜곡 안함.)
- **노버프 변환 불가**: 순수장비 스탯 구하려면 부위별 ilvl289 스탯 필요한데 Blizzard API는
  base ilvl(44)만 줌. ilvl스케일링은 시뮬수준이라 비현실 + 추측하면 부정확.
- **사용자 목적(절대 스탯 목표치)엔 풀버프가 정답**: 인게임서 음식·영약 켜고 스탯 맞추니
  WCL과 동일기준. 노버프(순수장비)는 실전에 안 나오는 수치라 목표로 무의미. → 현상태가 정확.
  UI에 "1~20평균=목표스탯, 풀버프기준, 음식영약 켜고 맞추면 됨" 명시함.

## 스펙 가이드 팝업 (data/spec_guide.json) — 꿀팁 누적
메타탭 스펙 클릭 팝업 = **좌(특징/지표) + 우(설명/로테/꿀팁)** 3구역. 우측은
data/spec_guide.json 키 "Class|Spec"(영문). **사용자가 영상 찾아주면 tips 배열에 누적**:
`{t:제목, d:내용(\n 줄바꿈), src:출처}`. main.py _spec_guide() 매요청 재로드(편집 즉시반영).
- **꿀팁 scope 필수**: tips 각 항목에 `scope:'쐐기'|'레이드'|'공용'` 달 것 (팝업서 배지로 구분).
  **사용자는 레이드 위주** → 레이드/공용 꿀팁이 중요, 쐐기전용은 본인한텐 덜 유용.
  예: 무리지도자 버프유지 매크로 = 쐐기전용(레이드는 전투 안 끊겨 불필요).
- **BM 등록됨**: 무리 지도자 버프유지 매크로(쐐기전용, Eyes of the Beast, Azortharion).
  나머지 스펙은 사용자가 영상 주면 추가 — 받으면 쐐기/레이드/공용 자동 분류해서 넣을 것.
